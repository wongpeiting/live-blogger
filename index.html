System Architecture

Parliament Live Blogging Pipeline

Stage 1 Listener Thread daemon · never blocks
▶ YouTube Live Stream URL ▶ transcript.jsonl
t=0s 🔗 yt-dlp Extract audio + video stream URLs from YouTube refreshes every 60min
→
t=0s 🎥 ffmpeg Start 30s audio capture via Popen (non-blocking) 16kHz mono PCM
→
t=5s 📸 ffmpeg Capture video frame (5s delay ensures speaker on-screen) 360p single frame
→
t=5s 👤 Face ID Match face against 108 MP encodings (128-dim, threshold 0.5) mp_faces/ → face_shots/
→
t=30s 🎤 Whisper Transcribe audio chunk (Singlish-trained small model) jensenlwt/whisper-small-singlish-122k
→
t=30s 🗣 pyannote Speaker diarisation — identify speaker segments by timecode pyannote/speaker-diarization-3.1
→
📄 Merge Combine transcript + speakers + identified_speaker → JSONL entry append to transcript.jsonl
transcript.jsonl  via threading.Event
Stage 2 Coordinator main thread · triage + dispatch
📄 Read Chunks Read new lines since cursor position from transcript.jsonl
→
⚒ Corrections Apply corrections.json: names → terms → abbreviations (longest-first) auto-reloads on mtime change
→
✨ Gemini 2.5 Flash Triage: score 1-5 newsworthiness (batched, max 8 chunks/call) low temp 0.3
→
📦 Dispatch Build batches by score rules, gather context radius, submit to pool
